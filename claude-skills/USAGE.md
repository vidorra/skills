# How to Use Your AI Skills

This guide explains how to effectively use your 12 specialized AI skills for UX/Mendix development, NextJS/SaaS development, and freelance business management.

## Quick Start

To use any skill, simply reference it in your conversation with Claude:

```
As a [Skill Name], help me with [task]
```

or

```
Using the [Skill Name] skill, [request]
```

**Example**:
```
As a Mendix UX Developer, help me customize the Atlas UI theme with custom SASS variables
```

## When to Use Which Skill

### Starting a New Project

**1. First Client Contact**
- **Skill**: English or Dutch Business Text Writer
- **Use for**: Professional introduction email
- **Example**: "As a Dutch Business Text Writer, help me write a follow-up email after meeting a potential client at a networking event"

**2. Contract Review**
- **Skill**: Contract Review & Legal Advisor
- **Use for**: Reviewing client's proposed contract
- **Example**: "As a Contract Review & Legal Advisor, review this MSA and identify any DBA risks and unfavorable terms"

**3. Project Scoping**
- **Skill**: Project Scoping & Estimation Specialist
- **Use for**: Understanding scope, estimating time and cost
- **Example**: "As a Project Scoping & Estimation Specialist, help me estimate a Mendix app with 10 entities, 25 pages, and basic CRUD functionality"

**4. Creating Quote**
- **Skills**: Project Scoping & Estimation + Dutch/English Business Text Writer
- **Use for**: Creating professional quote with accurate pricing
- **Example**: "Using Project Scoping & Estimation, calculate the project cost. Then as a Dutch Business Text Writer, format it into a professional quote"

### Design Phase

**5. User Research (if needed)**
- **Skill**: UX Researcher
- **Use for**: Creating personas, planning interviews, analyzing data
- **Example**: "Using the UX Researcher skill, help me create 3 customer personas based on this interview data"

**6. Wireframing & Design**
- **Skill**: UX Designer
- **Use for**: Figma designs, wireframes, prototypes, design systems
- **Example**: "As a UX Designer, help me organize my Figma file structure for a complex Mendix application"

**7. Design System (if needed)**
- **Skill**: Design System Specialist
- **Use for**: Creating HTML/CSS component libraries, design tokens
- **Example**: "Using the Design System Specialist skill, create a button component system with CSS variables for light/dark themes"

### Development Phase

**8. Mendix Development**
- **Skill**: Mendix UX Developer
- **Use for**: Building Mendix apps, theming, custom styling
- **Example**: "As a Mendix UX Developer, help me create custom design properties for card components with different variants"

**9. JavaScript Actions (when needed)**
- **Skill**: Mendix JavaScript Developer
- **Use for**: Client-side logic, browser APIs, complex validations
- **Example**: "As a Mendix JavaScript Developer, create a JavaScript Action that validates Dutch postal codes in real-time"

### Testing Phase

**10. Quality Assurance**
- **Skill**: Automation Tester
- **Use for**: Accessibility, performance, cross-browser testing
- **Example**: "As an Automation Tester, create a test plan for accessibility testing this Mendix app using free tools"

### Project Completion

**11. Client Communication**
- **Skill**: Dutch or English Business Text Writer
- **Use for**: Project updates, handoff emails, invoices
- **Example**: "As a Dutch Business Text Writer, create an email announcing project completion and sending the final invoice"

**12. Handling Scope Changes**
- **Skills**: Project Scoping & Estimation + Business Text Writer
- **Use for**: Quoting additional work, managing scope creep
- **Example**: "The client wants to add 5 more pages. As a Project Scoping & Estimation Specialist, calculate the additional cost. Then as a Dutch Business Text Writer, write an email explaining the scope change process"

**13. Payment Reminders (if needed)**
- **Skill**: Dutch or English Business Text Writer
- **Use for**: Friendly to assertive payment reminders
- **Example**: "Using the Dutch Business Text Writer skill, create a second payment reminder for an invoice that's 25 days overdue"

- ‚ùå **Don't use wrong skill**: Don't ask Mendix UX Developer for UX research help

## Troubleshooting

### "I'm not sure which skill to use"

**Solution**: Check the "When to Use Which Skill" section above or ask:
```
"Which skill should I use for [your task]?"
```

### "The skill is asking too many questions"

**Reason**: Some skills (like Project Scoping) need specific information for accuracy.

**Solution**: Provide as much detail as you can upfront, or answer the questions - it leads to better results.

### "Can I use multiple skills at once?"

**Yes!** Use them in sequence:
```
"First, as a Contract Review & Legal Advisor, review this contract.
Then, as a Dutch Business Text Writer, create a negotiation email based on the findings."
```

### "The output doesn't sound like me"

**For Business Writers**: These skills are calibrated to your style, but you can ask:
```
"Make this more direct" or "Make this more diplomatic"
```

**For Technical Skills**: Ask for adjustments:
```
"Explain this in simpler terms" or "Add more technical detail"
```

## Tips for Maximum Effectiveness

### 1. Start Conversations with Role

Always begin with "As a [Skill Name]..." so Claude knows which expertise to use.

### 2. Provide Context

Bad: "Help me with a contract"
Good: "As a Contract Review & Legal Advisor, review this 3-month Mendix development contract. I'm concerned about the liability clause and DBA risk. [paste contract]"

### 3. Ask for Specific Deliverables

Bad: "Help me estimate a project"
Good: "As a Project Scoping & Estimation Specialist, create a detailed quote for a 10-page Mendix app redesign with custom theming. Include work breakdown and pricing options."

### 4. Iterate and Refine

Don't accept first draft. Ask:
- "Make this more assertive"
- "Add more detail to the JavaScript Action error handling"
- "Explain the DBA risk more clearly"

### 5. Save Good Templates

When you get a great quote template, contract clause, or code snippet, save it for reuse.

### 6. Combine Skills for Complex Tasks

Most real work requires multiple skills. Don't hesitate to use 3-4 skills in one conversation.

## Getting the Most Value

### Use Skills Proactively

Don't wait for problems:
- Review EVERY contract (Contract Legal Advisor)
- Estimate EVERY project (Project Scoping & Estimation)
- Test accessibility EVERY project (Automation Tester)
- Set clear scope EVERY project (Project Scoping + Business Writer)

### Learn from Each Project

After completing a project, review:
- Was estimate accurate? (adjust buffers next time)
- Did scope creep happen? (tighten definitions)
- Were there contract issues? (add to red flag list)
- Did testing catch problems? (test earlier next time)

### Build Your Templates

Create a personal library:
- Contract negotiation emails that worked
- Quote templates for common project types
- Estimation spreadsheets for typical work
- Testing checklists
